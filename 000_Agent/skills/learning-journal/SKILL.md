# 每日學習日記（整合 /reflect）

用 5 個引導問題幫 ZY 寫下每天的學習紀錄。
問題設計依照「事實 → 感受 → 反思 → 改進 → 行動」的順序，引導從回顧走到規劃。

## 觸發條件

ZY 說以下任何一句話時觸發：
- 「寫日記」
- 「今天結束了」
- 「學習紀錄」
- 「journal」
- 「記錄一下今天」
- 「/reflect」
- 「反思」
- 「每日反思」

## 日記檔案位置

`e:/ZYoooClaude/journals/YYYY-MM-DD.md`

如果 `journals/` 資料夾不存在，自動建立：
```bash
mkdir -p "e:/ZYoooClaude/journals"
```

## 寫日記流程

### Step 0：確認日期與檢查既有日記

1. 先執行 `date` 確認今天的日期（台北時間）
2. 檢查 `e:/ZYoooClaude/journals/YYYY-MM-DD.md` 是否已經存在
   - **已存在**：告知 ZY「今天的日記已經有了，要**補充**到結尾，還是**重新寫**一篇？」
     - 補充 → 讀取現有日記，Step 1 讓 ZY 補充，最後合併寫回
     - 重新寫 → 從 Step 1 重新開始，覆蓋原檔案
   - **不存在**：繼續下一步

3. 讀取前一天的日記（如果存在）。
如果前一天的「明天要做的事」有內容，先口頭提醒 ZY：

> 昨天你列了這些想做的事：
> - [ ] xxxxxx
> - [ ] xxxxxx
>
> 等等寫日記時可以回顧一下有沒有做到。

### Step 1：引導問答（逐題問，不要一次全丟）

用 AskUserQuestion 工具逐題引導，每題等 ZY 回答後再問下一題。
ZY 說「跳過」就跳過該題。

**Q1：今天做了哪些事？發生了什麼？**
> 不管大小都算——裝了什麼、試了什麼、改了什麼、讀了什麼都可以說。

**Q2：有什麼讓你驚喜、覺得有趣的時刻？**
> 可能是 AI 做了一件讓你意想不到的事，或是你突然搞懂了一個觀念。

**Q3：哪些做得不錯？哪些可以更好？有卡住的地方嗎？**
> 做得好的值得記下來，卡住的也是。還沒解決沒關係，記錄下來之後可以回來看。

**Q4：明天我「能控制」並改進的一件小事是什麼？**
> 不用想很大的改變，一件小事就好。例如「下次先問 AI 再自己查」或「每做完一段就推上 GitHub」。

**Q5：明天最重要的三件事？**
> 不用想太多，列出來就好。沒想到可以跳過。

### Step 2：生成日記草稿並確認

根據 ZY 的回答，組裝成以下格式，語氣保留 ZY 原本的說話方式，不要過度潤飾：

```markdown
# YYYY-MM-DD 學習日記

## 今天做了什麼
（根據 Q1，條列式整理，保留原話，不要摘要）

## 驚喜時刻
（根據 Q2，保留語氣）

## 反思
（根據 Q3。做得好的和可以更好的都列。卡關用「問題 → 解法」格式）

## 明天的一個小改進
（根據 Q4，一句話）

## 明天要做的事
- [ ] 第一件事
- [ ] 第二件事
- [ ] 第三件事
```

規則：
- 保留 ZY 的原話和所有細節，不要摘要
- 跳過的題目不留空標題，直接省略
- 「明天要做的事」用 `- [ ]` checkbox 格式

把草稿展示給 ZY 看，問：
> 日記寫好了，有要修改的地方嗎？沒問題的話我就存檔了。

ZY 確認後才存檔。

### Step 3：存檔

用 Write 工具存入 `e:/ZYoooClaude/journals/YYYY-MM-DD.md`

### Step 4：推上 GitHub

存完後問：
> 要幫你推上 GitHub 嗎？養成習慣，每天存檔一次最安全。

ZY 同意就執行：
```bash
cd "e:/ZYoooClaude" && git add journals/YYYY-MM-DD.md && git commit -m "journal: YYYY-MM-DD 學習日記" && git push
```

## Session Wrap 暫存功能

當 ZY 說「要開新視窗」「重開對話」「換到下一個 session」，且完成 Session Wrap-up 後，自動將 Wrap 內容附加到當天的 `journals/YYYY-MM-DD.md`：

1. 檢查 `e:/ZYoooClaude/journals/YYYY-MM-DD.md` 是否存在
   - 存在 → 在檔案結尾追加以下區塊
   - 不存在 → 先建立空白日記，再追加

2. 追加格式：

```markdown
---

## Session Wrap — 視窗 N（YYYY-MM-DD HH:MM）

**完成的事**
- 條列本視窗完成的任務

**待解決**
- 條列未完成或下次繼續的事
```

3. 不需要 ZY 確認，直接寫入。

當天結束寫學習日記時（Step 1 問答**之前**），如果 `journals/YYYY-MM-DD.md` 已有 Session Wrap 區塊，執行以下流程：

**Step A：讀取並彙整所有 Session Wrap**

讀取當天日記中所有 `## Session Wrap` 區塊，整理成一份簡短的今日摘要，格式如下，展示給 ZY 看：

> 今天你有 N 個 Session Wrap 紀錄，以下是今天完成的事：
>
> - 條列所有視窗「完成的事」（合併去重）
>
> 還有這些待解決：
> - 條列所有視窗「待解決」的事
>
> 確認沒問題的話，我們開始今天的 5 個問題。

**Step B：等 ZY 確認後，才進入 Step 1 問答**

ZY 確認或說「沒問題」後，才開始逐題問。

**Step C：生成日記時整合所有內容**

在 Step 2 生成日記時，把 Session Wrap 的「完成的事」自動彙整進「今天做了什麼」，不重複貼原始 Wrap 區塊。

## 隔天開對話時的行為

當 ZY 開啟新對話時，如果前一天的日記存在且有「明天要做的事」，主動口頭提醒：

> 早安！昨天你列了這些今天想做的事：
> - [ ] xxxxxx
>
> 要從哪個開始？還是有新的計畫？

如果前一天有「明天的一個小改進」，也一起提醒：
> 還有，你昨天說今天想改進的一件事是：「xxxxxx」

## 注意事項

- 永遠用 `date` 取得真實時間，不要假設日期
- 問問題時語氣輕鬆自然，像朋友聊天
- 不要催促，尊重 ZY 的表達方式
- 若 ZY 中途說「不想寫了」或「跳過」，尊重並結束，不存檔
