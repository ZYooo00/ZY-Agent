## 壹、大原則

### 1.1 管理範疇

本系統涵蓋台北胚胎室**所有試劑與耗材**，包括：

- 培養液（Medium）：有效期、部分需 QC
- 試劑（Reagent）：有效期、部分需 QC
- 耗材（Consumables）：部分有效期、一般不需 QC

> 範疇與現行 Google Sheet「02. 台北胚胎室/精蟲室-所需訂購品」一致，不限縮為「培養液」。

### 1.3 批號追蹤原則

- 每次收貨記錄一筆，包含：品項名、訂貨人、訂貨數量、訂貨時間、收貨數量、收貨時間、產品批號、效期、收貨人
- 分批到貨時，每批各自新增一列（複製品項名，填各批數量與批號）

### 1.4 QC 管理原則

> 2026-04-29 更新：培養液類（GxTL、G-IVF、H5GT、EmbryoGlue）**目前全部免 QC**。

- **需要 QC 的品項**（到貨後不能立即使用，需先測試）：
  - **102（解凍試劑）**：需 QC，但以**月點料**方式管控（登記盒數），到貨時透過通知觸發 TE QC 備盤，不走每日追蹤邏輯
  - 其他視主管決定（品項主檔保留「QC 所需天數」欄位，預設 0）
- **QC 通過前禁止正式使用**（可標示備用）
- QC 截止日 = 上一批預計用完日 − QC 所需天數

---

### 1.5 效期管理原則

- **廠商效期**（未開封）：每批入庫時記錄，系統追蹤
- **開封後效期**：部分品項開封後有使用期限限制，需另外記錄開封日
- **Oil / HEPES 包裝單位說明（2026-04-30 確認）**：
  - **罐（小包裝）**：分裝後的使用單位（Oil ≈ 50 mL/罐；HEPES ≈ 50–55 mL/罐）
  - **瓶（大包裝）**：原裝庫存單位（500 mL/瓶）；備盤頁記錄「今日開封幾瓶」供庫存扣除
  - 1 瓶可分裝 Oil ≈ 10 罐、HEPES ≈ 9 罐
  - Oil 除分裝外，同時作為備盤覆蓋油使用
- HTF 現況：**停用**（SPS 停售，主力改用亞樸 G-IVF Plus；系統不納入）
- Geri ONE 現況：**停用**（系統不納入）

### 1.6 借出 / 還入原則

- 借給其他院區（或由其他院區歸還）時，以 **−/+ 數量** 新增一筆紀錄
- 同時填寫批號、效期、操作人，確保庫存帳面正確
- 借出視同「出庫」，歸還視同「入庫」

## 參、訂貨流程

### 3.1 現行流程

```
Step 1：人工盤點
  └─ 查冰箱、常溫各櫃子，確認各品項剩餘數量

Step 2：填訂購總表（Google Sheet）
  └─ 找到該品項的列，在對應月份欄填入需要訂購的數量
  └─ 若當月追加，填「XX 月追加」欄

Step 3：填叫貨量分頁
  └─ 依廠商分別填入本次實際要叫的數量
  └─ 本次不叫的品項填「-」

Step 4：確認生成訊息分頁
  └─ Apps Script 自動依叫貨量生成各廠商 LINE 訊息
  └─ 人工核對訊息內容是否正確

Step 5：送出（按「送出」按鈕）
  └─ 各廠商子分頁記錄訂貨人、訂貨數量、訂貨時間
  └─ 複製訊息 → 傳 LINE 給對應廠商業務
```

### 3.2 目標流程（系統輔助後的差異）

| 步驟 | 現行 | 目標 |
|---|---|---|
| 盤點 | 人工紙本 | 系統顯示目前庫存量 + 安全庫存比對 |
| 決定訂購量 | 看「基準量」人工判斷 | 系統推薦建議量（根據月用量 + 安全庫存） |
| 叫貨訊息 | 手動確認後複製 LINE | 可直接從系統產出訊息 |
| 記錄訂單 | 手動填 Sheet | 系統自動建立訂單記錄 |

## 肆、收貨 / 入庫流程

### 4.1 標準流程

```
Step 1：對照訂單
  └─ 確認是否為已下訂的貨

Step 2：確認數量
  └─ 清點實際到貨數量是否與訂購量相符

Step 3：記錄收貨資訊
  └─ 收貨數量、收貨時間、產品批號、效期、收貨人

Step 4：QC 標記（若適用）
  └─ 培養液 / 玻璃化試劑類：貼「待測」便條紙，不可立即投入使用
  └─ 亞樸品項：填「是否已測QC」欄位為「待測」

Step 5：通知相關人員
  └─ 收到新培養 Medium 需即時通報
```

### 4.2 分批到貨處理

- 若同一訂單分批到貨：
  1. 新增一列
  2. 複製品項名稱
  3. 填寫本次到的收貨數量、時間、批號、效期
- 效期比現有庫存舊 → 系統需提示警告

### 4.3 借出 / 還入

- 借出：新增一筆記錄，收貨數量填負數（如 -2），備注填「借給 XX 院區」
- 還入：新增一筆記錄，收貨數量填正數，補上批號與效期

### 5.2 QC 通過前的限制

- 貼「待測」便條紙
- 不得放入正式備盤選單
- QC 通過後：撕掉便條紙，填寫測試日期

### 5.3 QC 截止日推算

```
最晚 QC 截止日 = 上一批預計用完日 − QC 所需天數
上一批預計用完日 = 開始使用日 + (剩餘量 ÷ 日均消耗量)
```

```

  └─ 兩人同步配置 Medium，核對盤數與紙條一致
```

## 柒、訂購前資訊顯示 + 補貨邏輯

### 7.1 補貨模型（ZY 確認採用）

```
本次訂購量 = 目標庫存量 − 目前盤點量（點料時眼睛看到的數量）
```

- 點料時只需記錄**當下架上看到的實際數量**
- 系統自動計算建議訂購量，可手動調整後才送出
- 若結果 ≤ 0，表示庫存充足，本次不需訂購
- 耗材採此模型；培養液待備盤系統上線後可切換為實際消耗量

### 7.2 目標庫存量（Target）的設定策略

| 階段 | 時間點 | 做法 |
|---|---|---|
| Phase 1（初期）| 系統建立時 | 由有經驗的人員手動輸入基本值（依經驗判斷）|
| Phase 2（精進）| 每季 / 每半年 | 依歷史訂購紀錄 + 消耗資料，建構更精準的預測模型 |

**Target 計算基礎（供參考）：**
```
目標庫存量 = 月均用量 × （廠商交期月數 + 安全緩衝月數）
```

範例（GxTL，亞樸）：月均用量 ≈ 19 瓶，交期 2 週，安全緩衝 1 週 → 目標 ≈ 14 瓶

## 捌、訂貨提醒與預留時間

### 8.1 觸發邏輯

```
觸發條件：剩餘可用天數 ≤ 廠商交期天數 + QC 天數 + 安全緩衝天數

剩餘可用天數 = 現有庫存 ÷ 日均消耗量
日均消耗量 = 近 3 個月訂購量 ÷ 90 天（初期以訂購量估算）
```

### 8.2 各廠商提醒提前天數

| 廠商 | 交期 | QC 天數 | 建議提醒提前天數 |
|---|---|---|---|
| 明美 | 4-5 天 | 0 | 14 天 |
| 億宸 | 1–2 週 | 0 | 21 天 |
| 弘優 | 1–2 週 | 0 | 21 天 |
| 亞樸 | 約 3 週 | 0（目前培養液免 QC）| 28 天 |
| 磊柏 | 約 3 週 | 102 需 QC（天數待確認）| **45 天**（交期長，需最早提醒）|

### 8.3 通知方式

- **LINE 推播**：觸發時主動發送，對象：訂貨負責人 + 主管
- **系統儀表板**：顯示紅色（緊急）/ 橘色（預警）分級警示
- 磊柏品項單獨標為「橘色預警」（交期最長，需早處理）

### 8.4 提醒設計原則

1. **防止告警疲勞**：同一品項提醒後若 N 天未處理，才重複推播
2. **加「已訂貨」確認按鈕**：確認後停止提醒
3. **同廠商品項合併為一則**：5 個磊柏品項低庫存 → 1 則摘要
4. **「本月訂貨負責人」需要設定**：每月初填入，推播才知道發給誰

## 拾、製程週期訂貨提醒（固定製程週期品項）

### 10.1 背景說明

部分培養液廠商採**固定週期製程**生產，每批有對應的製作日期與效期。
廠商通常**每年提供一次年度製程時程表**。

目前已知有此特性的品項：
- **H5GT**（Life Global）
- **GxTL**（Vitrolife）
- **G-IVF**（Vitrolife）

> 此功能設計為通用架構：只要某品項有固定製程週期，就可以建立對應的年度時程表。

### 10.2 製程時程表資料結構

| 欄位 | 說明 | 範例 |
|---|---|---|
| 品項 | 培養液名稱 | H5GT |
| 廠商 | 供應商 | 弘優 / Life Global |
| 製作日 | 廠商預計製造日 | 2026-05-12 |
| 預計到貨日 | 製作完成 + 運送時間 | 2026-05-26 |
| 廠商效期 | 批次有效期限 | 2026-09-30 |
| **訂貨截止日** | 最晚必須下單的日期 | 2026-04-28 |
| 提醒狀態 | 未提醒 / 已提醒 / 已訂貨 | — |

> 訂貨截止日 = 製作日 − 廠商需要的前置訂單天數（各廠牌不同，需確認）

### 10.3 年度製程表管理流程

```
每年（廠商提供製程時程表時）
  └─ 將時程表輸入系統（手動逐筆 or 匯入）
  └─ 系統自動建立全年的訂貨截止日提醒清單

每批訂貨截止日前 N 天
  └─ 系統推播 LINE：「H5GT 本批訂貨截止日 XX/XX，請確認是否下單」
  └─ 負責人確認後點選「已訂貨」，提醒停止

收到此批貨時
  └─ 在收貨記錄中對應批次，確認批號與效期是否一致
```

### 10.4 與一般庫存低警示的差異

| | 一般庫存低警示 | 製程週期訂貨提醒 |
|---|---|---|
| 觸發來源 | 庫存量動態計算 | 年度製程時程表（日曆型）|
| 提醒時機 | 動態（視消耗速度）| 固定（每批截止日前）|
| 適用品項 | 所有品項 | 有固定製程週期的品項 |
| 資料更新 | 系統即時 | 每年人工更新一次 |

> 兩種提醒機制**並行運作**，互不取代。

## 陸、備盤 / 估盤規則

> 更新：2026-04-29 ZY 確認

### 6.1 FBT / BT / TBx 盤子備盤規則

**這是估盤人需要知道的規則，決定每日要備哪些胚胎盤種類。**

| 狀況 | Glue（EmbryoGlue）盤 | TB（滋養層活檢）盤 |
|---|---|---|
| FBT（冷凍胚胎植入，非 TBx）| ✅ 備 | ❌ 不備 |
| BT（新鮮胚胎植入）| ✅ 備 | ❌ 不備 |
| TBx（滋養層切片後植入）| ❌ 不備 | ✅ 備 |

> FBT 和 BT 患者需要使用 EmbryoGlue 培養液，因此備 Glue 盤；TBx 患者需要進行切片，因此只備 TB 盤，不需要 Glue 盤。

### 6.2 多配（多倍體）體積規則

- 多配**不以盤數計算**，以**體積（mL）計算**
- 每批培養液最少保留 **2 cc** 作為多配用途（系統自動估算）
- 適用培養液：**G-IVF、GxTL、H5GT** 皆適用
- 估盤時系統自動計算建議多配體積，估盤人不需另外填多配盤數
- 備盤頁面的「多配 (mL)」欄位顯示系統估算值，備盤人可手動調整
- 多配 mL 調整時，右欄對應試劑的「今日需用」即時重算（2026-04-30 prototype 確認）

### 6.3 AOA 盤分拆規則（2026-04-30 確認）

AOA 盤分為兩種，備盤表中需分別記錄：

| 盤子名稱 | 對應試劑 | 說明 |
|---|---|---|
| AOA 弘優盤 | AOA 弘優 CI（CIR）| 常規 AOA 患者 |
| AOA 508 盤 | AOA 明美 GM508 | 特定客戶（需填病歷號）|

> 兩種 AOA 盤使用不同試劑，必須在估盤時分別填入台數，系統才能計算各自的試劑需用量。
